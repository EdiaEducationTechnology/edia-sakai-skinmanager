The Sakai Skin manager comes in binary or source distribution, the source distribution needs to be build from within the Sakai source tree, the binary distribution can be installed an existing Sakai installation directly.

Prerequisites

This manual assumes that you are familiar with installing, building, and running Sakai. You must be familiar with the concepts in the Sakai installation guide, which can be found at:

  • 2.2.1 Installation Guide
  • 2.3.1 Installation Guide

Of course, you need a Sakai installation to use this Sakai tool.

Installing from binaries

Downloading the binary package

The skin manager binary package can be downloaded for the following versions at the following locations.

Sakai version Location
2.2.x skinmanager-0.0.1-sakai-2.2.1.zip
2.3.x skinmanager-0.0.1-sakai-2.3.1.zip

Installing the binary package

The binary version of the tool must be unpacked in the Sakai installation root. We strongly advise you stop Sakai before installing, the tool contains a component that configures at Sakai startup. The skin manager does not require any changes to the Sakai configuration, except for the auto.dll property enabled on the first run to create the database table, see the database section for details. Users need to have administrator privileges or the specific Sakai Skin Manager permissions set to be able to use the tool.

Installing from source

Downloading the source

The source can be downloaded from the following location:

Sakai version Location
2.2.x/2.3.x skinmanager-src-0.0.1.zip

Installing the source package

The source version of the Sakai Skin manager must be unpacked in the source root of Sakai. If you run an entire build on Sakai, it will automatically be part of that build. But if the source tree has already be build, it can be build individually by running a build in the following order:

  1. edia-sakai-utils
  2. edia-sakai-skin-manager

Of course, to do the build the sakai target needs to be be used:

maven sakai

Note: the ibiblio repository has been restructured, and maven 1.x has trouble understanding the http return code 301 (moved permanently). For that reason, it might be necessary to add the following to the maven.repo.remote property in the build.properties located in the home dir:

http://repo1.maven.org/maven/

Database model

The Sakai skin manager makes use of one table, which is mapped by hibernate. Sakai needs to be started with the

auto.ddl=true

set in the sakai.properties on the first run of the skin manager to let hibernate create the required table for you.
